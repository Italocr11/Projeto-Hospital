# Requisitos Funcionais 
### Sistema de Gestão Hospitalar<br>
Este documento apresenta os requisitos funcionais do sistema de gestão hospitalar, descrevendo as funcionalidades que o sistema deve oferecer.

`RF01` - **Cadastro de Pacientes**<br>
O sistema deve permitir o cadastro de pacientes com seus dados pessoais.

`RF02` - **Cadastro de Médicos**<br>
Descrição: O sistema deve permitir o cadastro de médicos com suas especialidades.

`RF03` - **Cadastro de Funcionários**<br>
O sistema deve permitir o cadastro de funcionários do hospital.

`RF04` - **Registro de Consultas**<br>
O sistema deve registrar consultas realizadas entre pacientes e médicos.

`RF05` - **Agendamento de Consultas**<br>
O sistema deve permitir o agendamento de consultas em datas e horários específicos.

`RF06` - **Registro de Internações**<br>
O sistema deve registrar internações de pacientes.

`RF07` - **Controle de Leitos**<br>
O sistema deve controlar a ocupação e disponibilidade dos leitos.

`RF08` - **Registro de Exames**<br>
Descrição: O sistema deve registrar exames solicitados pelos médicos.

`RF09` - **Registro de Resultados de Exames**<br>
O sistema deve armazenar os resultados dos exames realizados.

`RF10` - **Registro de Prescrições**<br>
O sistema deve registrar prescrições médicas.

`RF11` - **Cadastro de Medicamentos**<br>
O sistema deve permitir o cadastro de medicamentos.

`RF12` - **Gerenciamento de Setores**<br>
O sistema deve permitir o cadastro e gerenciamento dos setores do hospital.

`RF13` - **Associação com Convênios**<br>
O sistema deve permitir vincular pacientes a convênios.

`RF14` - **Cadastro de Responsáveis**<br>
O sistema deve permitir o cadastro de responsáveis por pacientes.

`RF15` - **Vinculação de Médicos a Setores**<br>
O sistema deve permitir associar médicos aos setores do hospital.
